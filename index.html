Es mucho más rápido y seguro. Sigue estos pasos: 1. Abre el SQL Editor Ve al icono de la consola (>_) en la barra lateral izquierda de Supabase. 2. Ejecuta este comando Copia y pega el siguiente código. Este script busca al usuario por su correo y le inyecta el metadato de "admin" directamente en la tabla interna de autenticación de Supabase. SQL -- Asignar rol de ADMIN al usuario específico UPDATE auth.users SET raw_user_meta_data = jsonb_set( COALESCE(raw_user_meta_data, '{}'::jsonb), '{role}', '"admin"' ) WHERE email = 'sjh110125@gmail.com'; -- Asignar rol de USUARIO al otro (opcional, para estar seguros) UPDATE auth.users SET raw_user_meta_data = jsonb_set( COALESCE(raw_user_meta_data, '{}'::jsonb), '{role}', '"user"' ) WHERE email = 'lordgrimm22@gmail.com'; 3. Verifica el cambio Una vez que le des a "Run", vuelve a la pantalla de Authentication > Users, haz clic en el usuario y ahora en la pestaña Raw JSON deberías ver algo como esto: JSON "raw_user_meta_data": { "email_verified": true, "role": "admin" } ¿Por qué hacerlo así? Supabase protege mucho la tabla de usuarios. A veces, dependiendo de los permisos de tu cuenta, la interfaz gráfica bloquea la edición de metadatos para evitar errores accidentales. El SQL Editor salta esa restricción de la interfaz y escribe directamente en el "cerebro" del sistema de usuarios. ¿Qué pasa con el Dashboard?
